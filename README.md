# QQSpeedM-Lab (QQ飞车手游实验室)

`qqspeedm-lab` 是一个专门为《QQ飞车手游》设计的分析工具。目前实现的功能主要是能够通过 OCR（光学字符识别）技术从游戏录像中提取实时速度数据，并利用稳健回归算法（RANSAC）分析赛车的加速度、初速度等核心动力参数。

## 🌟 核心功能

* **设备实时录屏**：集成 `scrcpy` 控制，支持在应用内直接启动、停止 Android 设备录屏并自动保存。
* **精准视频切片**：利用 FFmpeg 毫秒级裁剪视频，快速锁定需要分析的路段。
* **交互式 ROI 提取**：
    * **预设坐标**：支持针对指定场景的一键 ROI 区域选择。
    * **手动框选**：提供基于 `streamlit-drawable-canvas` 的交互式画布，支持两步放大框选，精准定位速度数字区域。
* **稳健 OCR 识别**：
    * 基于 `CnOcr` 定制识别引擎，针对数字和小数点进行优化。
    * 内置图像预处理（灰度化、二值化），提升复杂背景下的识别准确率。
* **动力学深度分析**：
    * **RANSAC + OLS 线性拟合**：自动剔除 OCR 误读产生的离群点（Outliers），获取高精度的平均加速度 $a$ 和拟合优度 $R^2$。
    * **Jerk（加加速度）趋势分析**：通过对速度微分并平滑处理，检测斜率变化趋势（如检测赛车是否处于加速越来越快的状态）。
* **自动化实验管理**：分析完成后，系统会自动保存原始数据 (CSV)、实验参数 (YAML) 以及高质量的 V-T/A-T 矢量图表 (PDF)。

## 🏗️ 项目架构

项目采用模块化设计，核心逻辑与 UI 界面解耦：

* **`core/` (核心逻辑层)**
    * `kinematics.py`: 运动学分析核心，包含 RANSAC 拟合与绘图逻辑。
    * `ocr.py`: OCR 处理器，负责 ROI 裁剪与文字识别。
    * `video.py`: 视频处理封装，负责元数据获取、切片及帧提取。
    * `data_service.py`: 数据持久化服务，管理实验结果的保存与合并编辑。
    * `recorder.py`: 基于 `scrcpy` 的录屏包装器。
* **`ui/` (Streamlit 表现层)**
    * `main.py`: 整个视频处理管线的调度中心。
    * `slicer.py` & `roi.py`: 视频剪辑与 ROI 选择交互界面。
    * `ocr.py` & `kinematics.py`: 数据识别编辑与动力学分析可视化组件。

## 🚀 环境要求与安装

### 1. 外部依赖
* **Python 3.8+**
* **FFmpeg**: 需添加至系统环境变量，用于视频切片。
* **Scrcpy**: 如需使用录屏功能，请确保已下载 `scrcpy` 并在 `config.yaml` 中配置其路径。

### 2. Python 依赖项
项目核心依赖及其版本如下（根据您的环境导出）：

| 关键包名 | 版本 | 用途 |
| :--- | :--- | :--- |
| `streamlit` | 1.29.0 | 基础 UI 框架 |
| `cnocr` | 2.3.2.3 | 文字识别引擎 |
| `torch` | 2.10.0 | 深度学习后端 |
| `scikit-learn` | 1.8.0 | RANSAC 稳健回归算法 |
| `pandas` | 2.3.3 | 数据处理与 CSV 导出 |
| `opencv-python`| 4.13.0.92 | 视频解码与图像处理 |
| `plotly` | 6.6.0 | 交互式图表渲染 |

## 🛠️ 使用流程

1.  **启动应用**：执行 `streamlit run app.py`。
2.  **选择视频**：在侧边栏浏览并选择待分析的文件。
3.  **视频切片**：设定起止时间，利用 FFmpeg 提取关键片段。
4.  **框选 ROI**：通过预设或手动绘图，指定速度数值在画面中的坐标。
5.  **OCR 识别**：系统自动扫描视频并提取数值，支持在表格中手动校对识别结果。
6.  **动力学分析**：系统自动生成拟合曲线。若 $R^2$ 达标，实验数据将自动保存至 `experiment_results/`。
